Основи Git та систем контролю версій

Системи контролю версій

Системи контролю версій (VCS) – це програмне забезпечення для управління змінами в документах, коді або будь-яких інших даних з можливістю зберігання історії версій та відкату до попередніх станів. VCS використовуються для співпраці та забезпечення цілісності даних.

Основні можливості Git

Git – це одна з найпопулярніших систем контролю версій, що має такі можливості:

  • Децентралізоване управління версіями.
  • Зберігання всієї історії проєкту локально.
  • Підтримка гілок та злиття коду (branching).
  • Висока швидкість роботи.
  • Можливість працювати в автономному режимі.

10 базових команд Git

  1. git init – Ініціалізація нового Git репозиторію.
  2. git clone <url> – Клонування існуючого репозиторію.
  3. git status – Перегляд поточного стану репозиторію.
  4. git add <file> – Додавання файлу до індексу для коміту.
  5. git commit -m "коментар" – Збереження змін з коментарем.
  6. git log – Перегляд історії комітів.
  7. git branch – Перегляд, створення або видалення гілок.
  8. git checkout <branch> – Перемикання на іншу гілку.
  9. git merge <branch> – Злиття змін з іншої гілки.
  10. git push – Надсилання змін на віддалений репозиторій.
